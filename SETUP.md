# Silicon to Software (S2S) — fresh-machine setup

After `git clone` on a new laptop, do these things **before** the first
`uvicorn main:app` run.

## 1. Python environment

```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# macOS / Linux:
source venv/bin/activate

pip install -r requirements.txt
```

Python 3.11 is what the project is tested against (older 3.10 should
work; 3.12 is fine too).

## 2. Environment file

Copy the template and fill in the keys you have:

```bash
# Windows:
copy .env.example .env
# macOS / Linux:
cp .env.example .env
```

Open `.env` and set:

| Var | Required for | Notes |
|---|---|---|
| `ANTHROPIC_API_KEY` or `GLM_API_KEY` | All AI phases | At least one. The pipeline falls back across providers when one rate-limits. |
| `GITHUB_TOKEN` | P8c auto-PR | PAT must have **both** `repo` AND `workflow` scopes (P8c writes `.github/workflows/*.yml`). |
| `GITHUB_REPO` | P8c auto-PR | Format `owner/repo`. Repo must already exist. |
| `GIT_ENABLED` | P8c auto-PR | Defaults to `true` when `GITHUB_TOKEN` is set. Set to `false` to skip git. |
| `GIT_BYPASS_CREDENTIAL_HELPER` | P8c on Windows / macOS | **Leave at default `true`.** Disables Git Credential Manager interception. |
| `DIGIKEY_CLIENT_ID` / `DIGIKEY_CLIENT_SECRET` | Component search (optional) | Falls back to ChromaDB if absent. |
| `MOUSER_API_KEY` | Component search (optional) | Same. |

## 3. Pandoc (for DOCX export)

The DOCX render path uses `pandoc`. The repo ships a Windows binary
at `bin/pandoc.exe` (gitignored — see step 4 if missing).

```bash
# verify
pandoc --version
# Windows fallback if PATH doesn't have it:
.\bin\pandoc.exe --version
```

The agent looks in `bin/pandoc.exe` first, then falls back to `PATH`.
On Linux/macOS:

```bash
sudo apt install pandoc      # Debian/Ubuntu
brew install pandoc          # macOS
```

## 4. (Optional) Re-download bundled binaries

`bin/pandoc.exe` is **gitignored** because it's 231 MB. On a fresh
clone you'll need to download it:

- Windows: https://github.com/jgm/pandoc/releases/latest → grab the
  `pandoc-X.Y.Z-windows-x86_64.zip` and extract `pandoc.exe` into
  `bin/`.
- Linux/macOS: `apt install pandoc` / `brew install pandoc` is enough
  (no `bin/` copy needed — agent finds it on `PATH`).

## 5. (Optional) Sanitise existing project outputs

If you cloned an `output/` folder generated by an OLDER agent, the
markdown files inside may have malformed mermaid blocks (bracket
mismatches, nested `[1:4]` inside labels, etc.) that fail to render
in the browser even though the new agent code would have fixed them.

Run the one-shot sanitiser to fix all existing files in place:

```bash
# Dry-run first to see what would change:
python scripts/sanitise_existing_outputs.py --dry-run

# Apply:
python scripts/sanitise_existing_outputs.py

# Sanitise a single project:
python scripts/sanitise_existing_outputs.py output/rx_band
```

The script walks every `output/*/*.md`, runs each fenced ` ```mermaid `
block through the deterministic coerce-and-re-render pipeline, and
writes the cleaned file back. Idempotent — running it twice produces
the same result.

## 6. Start the backend

```bash
uvicorn main:app --reload --port 8000
```

Then open `http://localhost:8000/app` in a browser.

## 7. Frontend

The React build is **already bundled** at `frontend/bundle.html` and
served at `/app`. Only rebuild if you've edited TS/CSS:

```bash
cd hardware-pipeline-v5-react
npm install
npx vite build
cd ..
python bundle_and_escape.py     # writes frontend/bundle.html
```

After ANY backend code change (agents, services, config) you must
restart `uvicorn` — `--reload` watches Python files but doesn't always
pick up new imports cleanly.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Mermaid "Syntax error" or "Parse error" in any document | Output file pre-dates a coerce fix | Run `scripts/sanitise_existing_outputs.py` |
| P8c always failed in 3-30 s with no PR | Stale dependency-graph (P26 #12) | Pull latest, restart backend |
| P8c shows "no GitHub remote configured" but env is set | GitHub Credential Manager intercepted the push | `.env` should have `GIT_BYPASS_CREDENTIAL_HELPER=true` (default), restart backend |
| P8c push fails with "workflow scope" | PAT missing `workflow` scope | Re-issue PAT with both `repo` AND `workflow` ticked |
| Phase appears completed instantly with no elapsed counter | Old interlude config (P26 #20 bumps to 8 s) | Pull latest, restart backend |
| SDD takes 12+ min and produces only template fallback | Server running pre-P26 #16 / #20 code | Pull latest, restart backend (legacy SDD loop now caps at 5 min) |
