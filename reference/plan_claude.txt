## API Integration

**Base URL:** `http://localhost:8000`
**API prefix:** `/api/v1/`
**Swagger docs:** `http://localhost:8000/docs`
**CORS:** Configured in FastAPI backend

| Method | Endpoint | Used For |
|--------|----------|----------|
| GET | `/api/v1/projects` | List all projects |
| POST | `/api/v1/projects` | Create new project |
| GET | `/api/v1/projects/{id}` | Project detail |
| GET | `/api/v1/projects/{id}/status` | All phase statuses — returns `{ phase_statuses: {...} }` |
| POST | `/api/v1/projects/{id}/pipeline/run` | Start full pipeline |
| POST | `/api/v1/projects/{id}/phases/{phase_id}/execute` | Run a single phase |
| POST | `/api/v1/projects/{id}/chat` | P1 design chat — returns full JSON (not a stream) |

**Create Project payload:** `{ name, description, design_type }` — do NOT include `product_id`, backend ignores/rejects it.

**Chat response:** Backend returns `{ response: "..." }` as a complete JSON object, not a streaming response. Display the full text at once.

**Status polling:** Phase status auto-refreshes every 3 seconds when a phase is running.

## Bundle & Deploy

### Bundle Script
After `npx vite build`, run this Python script to inline CSS + JS into a single HTML and escape all non-ASCII characters:

```python
# bundle_and_escape.py
import re, pathlib

dist = pathlib.Path("hardware-pipeline-v5-react/dist")
src_html = (dist / "index.html").read_text(encoding="utf-8")

# Inline CSS
for css_file in dist.glob("assets/*.css"):
    tag = f'<link rel="stylesheet" crossorigin href="/assets/{css_file.name}">'
    src_html = src_html.replace(tag, f"<style>{css_file.read_text('utf-8')}</style>")

# Inline JS
for js_file in dist.glob("assets/*.js"):
    tag = f'<script type="module" crossorigin src="/assets/{js_file.name}"></script>'
    js = js_file.read_text("utf-8")
    src_html = src_html.replace(tag, f'<script type="module">{js}</script>')

# CRITICAL: Escape all non-ASCII chars in <script> blocks
# Without this, Windows cp1252 encoding will crash the server with UnicodeDecodeError
def escape_script(m):
    content = m.group(1)
    escaped = re.sub(r'[^\x00-\x7F]', lambda c: f'\\u{ord(c.group()):04X}', content)
    return f'<script type="module">{escaped}</script>'

src_html = re.sub(r'<script type="module">([\s\S]*?)</script>', escape_script, src_html)

out = pathlib.Path("frontend/bundle.html")
out.parent.mkdir(exist_ok=True)
out.write_text(src_html, encoding="ascii")
print(f"Bundle written: {out} ({out.stat().st_size // 1024} KB)")
```

**Output:** `frontend/bundle.html` (~300 KB, ASCII-safe)

### FastAPI /app Route (in main.py)
```python
@app.get("/app", response_class=HTMLResponse, tags=["ops"])
async def serve_frontend():
    """Serve the React v5 frontend at http://localhost:8000/app"""
    import pathlib
    p = pathlib.Path(__file__).parent / "frontend" / "bundle.html"
    if p.exists():
        return HTMLResponse(content=p.read_text(encoding="utf-8", errors="replace"), status_code=200)
    return HTMLResponse(content="<h1>Frontend not built yet.</h1>", status_code=404)
```

## Critical Gotchas

**1. Windows cp1252 encoding crash**
The bundle.html must have ALL non-ASCII characters escaped as `\uXXXX` JavaScript escape sequences. Characters like the lightning bolt (U+26A1), bullet circle (U+25CF), em-dash (U+2014), etc. have byte values undefined in Windows cp1252. Python's `read_text()` will throw `UnicodeDecodeError` and serve a 500 error. The bundle script above handles this automatically.

**2. Do NOT open bundle.html as file://**
`type="module"` inline scripts block when opened via `file://` URL due to CORS restrictions. Always access via `http://localhost:8000/app`.

**3. FastAPI runs on port 8000, not 8001**
The API prefix is `/api/v1/` — always check `http://localhost:8000/docs` for exact endpoint signatures.

**4. Chat is not a streaming endpoint**
The `/api/v1/projects/{id}/chat` endpoint returns a complete JSON response, not a server-sent event stream. Display the full `response` field at once.

**5. No product_id in create project**
The backend `POST /api/v1/projects` does not accept a `product_id` field. Only send `name`, `description`, `design_type`.

**6. Component files must use default exports**
`Topbar`, `LeftPanel`, `RightPanel` and all view files must use `export default`. Named-only exports cause module resolution errors after bundling.

**7. uvicorn requires manual restart**
When adding new routes to `main.py`, the server must be restarted manually if not running with `--reload`.
